REST API routes Contacts¶

This module defines API routes for managing contact information. Routes include CRUD operations on contacts with rate limiting and user authentication.

async src.routes.contacts.create_contact(body: ContactSchema, db: AsyncSession = Depends(get_db), user: User = Depends(current_user_dependency))¶

Create a new contact for the authenticated user.

  • body: Data for the new contact.

  • db: Asynchronous database session.

  • user: Current authenticated user.

Returns the created contact. Rate limited to 1 request per 20 seconds.

  • contact_id: ID of the contact to delete.

REST API routes Users¶

This module defines API routes related to user operations such as: - Requesting email confirmation - Uploading and updating user avatar using Cloudinary

async src.routes.user.request_email(body: RequestEmail, background_tasks: BackgroundTasks, request: Request, db: AsyncSession = Depends(get_db))¶

Initiates the email confirmation process for a user.

  • body: Contains the email address to confirm.

  • background_tasks: Background task handler for sending emails asynchronously.

  • request: The incoming HTTP request object (used for building URL).

  • db: Asynchronous database session.

If email already confirmed, returns message; otherwise sends confirmation email.

async src.routes.user.upload_avatar(file: UploadFile = File(PydanticUndefined), user: User = Depends(current_user_dependency), db: AsyncSession = Depends(get_db))¶

Uploads and updates the user’s avatar image using Cloudinary.

  • file: Uploaded image file.

  • user: Current authenticated user.

  • db: Asynchronous database session.

Uploads avatar to Cloudinary, updates user’s avatar URL in the database, and returns the updated user data. Rate limited to 1 request per 20 seconds.

REST API routes Auth¶

This module defines API routes for user authentication and user management. It integrates FastAPI Users’ authentication system using JWT, user registration, password reset, email verification, and user CRUD operations.
